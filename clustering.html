Cap√≠tulo 9: Algoritmos de Clustering

¬øQu√© es el Clustering?

El clustering es una t√©cnica de aprendizaje no supervisado que agrupa datos similares en conjuntos llamados clusters. A diferencia del aprendizaje supervisado, el clustering no requiere datos etiquetados y busca patrones intr√≠nsecos en los datos para identificar grupos naturales.

Comparaci√≥n visual de diferentes algoritmos de clustering en conjuntos de datos simulados (Fuente: Scikit-learn)

El clustering se utiliza en numerosos campos y aplicaciones:

  • Marketing: Segmentaci√≥n de clientes basada en comportamiento de compra
  • Biolog√≠a: Agrupaci√≥n de genes con expresi√≥n similar
  • Astronom√≠a: Clasificaci√≥n de estrellas y galaxias
  • Medicina: Identificaci√≥n de subtipos de enfermedades
  • An√°lisis de documentos: Organizaci√≥n tem√°tica de textos
  • Detecci√≥n de anomal√≠as: Identificaci√≥n de patrones at√≠picos

Principales T√©cnicas de Clustering

Existen varios enfoques para el clustering, cada uno con sus propias fortalezas y caracter√≠sticas:

Clustering Basado en Centroides

Los algoritmos basados en centroides dividen los datos en K grupos, donde cada grupo est√° representado por un punto central llamado centroide.

El algoritmo m√°s conocido es K-means, que:

  • Asigna cada punto al centroide m√°s cercano
  • Recalcula los centroides como el promedio de los puntos asignados
  • Repite el proceso hasta la convergencia

Clustering Jer√°rquico

El clustering jer√°rquico crea una descomposici√≥n anidada de los datos, formando un √°rbol de clusters (dendrograma).

Existen dos enfoques principales:

  • Aglomerativo: Comienza con cada punto como un cluster y los fusiona gradualmente
  • Divisivo: Comienza con un solo cluster y lo divide recursivamente

Clustering Basado en Densidad

Los algoritmos basados en densidad identifican regiones densas de puntos separadas por regiones de baja densidad.

El algoritmo m√°s popular es DBSCAN (Density-Based Spatial Clustering of Applications with Noise), que:

  • Define clusters como √°reas densas conectadas
  • Puede detectar clusters de forma arbitraria
  • Identifica autom√°ticamente puntos de ruido o valores at√≠picos
A diferencia de la clasificaci√≥n y regresi√≥n, en el clustering no hay una "respuesta correcta" predefinida. La evaluaci√≥n de los resultados a menudo requiere m√©tricas internas (como cohesi√≥n y separaci√≥n) o validaci√≥n basada en conocimiento del dominio.

Para evaluar la calidad del clustering, definimos una funci√≥n que calcula m√∫ltiples m√©tricas de evaluaci√≥n:

# Funci√≥n para calcular m√©tricas de evaluaci√≥n
def evaluate_clustering(true_labels, predicted_labels, data):
    """
    Calcula las m√©tricas de evaluaci√≥n para clustering
    """
    silhouette = metrics.silhouette_score(data, predicted_labels)
    adjusted_rand = metrics.adjusted_rand_score(true_labels, predicted_labels)
    homogeneity = metrics.homogeneity_score(true_labels, predicted_labels)

    return {
        'Silhouette Score': silhouette,
        'Adjusted Rand Score': adjusted_rand,
        'Homogeneity Score': homogeneity
    }

# Evaluar KMeans

7.1 Optimizaci√≥n de K-Means

# Optimizaci√≥n de hiperpar√°metros para KMeans
print("OPTIMIZACI√ìN DE HIPERPAR√ÅMETROS - KMEANS")
print("=" * 50)

# Par√°metros a probar
kmeans_params = {'n_clusters': [2, 3, 4, 5]}
parameter_grid = ParameterGrid(kmeans_params)

best_kmeans_score = -1
best_kmeans_params = None
kmeans_results_opt = []

for params in parameter_grid:
    model = KMeans(random_state=42, max_iter=1000, **params)
    model.fit(pca_features)

    silhouette = metrics.silhouette_score(pca_features, model.labels_)

    result = {'params': params, 'silhouette_score': silhouette}
    kmeans_results_opt.append(result)

    print(f"Par√°metros: {params} | Silhouette Score: {silhouette:.4f}")

    if silhouette > best_kmeans_score:
        best_kmeans_score = silhouette
        best_kmeans_params = params

print(f"\nMejores par√°metros KMeans: {best_kmeans_params}")
